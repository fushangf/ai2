# 比赛演示脚本

## 3 分钟版本

### 0:00–0:20 赛前自检

进入工作台后说：

> 系统自检。

页面与语音会报告浏览器语音能力、麦克风、后端、数据库、AI 配置和本地路由状态。说明这是为了降低现场环境不确定性。

### 0:20–1:05 复杂创作

说：

> 开始，画一幅夜晚海边，月亮倒映在海面，远处有灯塔和三只海鸥，整体使用深蓝色调。

停止说话。指向音量条、三秒倒计时、AI 执行路径、拆解步骤与逐步绘制。

### 1:05–1:40 低延迟编辑与对象消歧

说：

> 开始，把月亮向右移动八十像素，然后把最左边的云朵变成粉色，再删除一只海鸥。

说明：确定性编辑无需远程模型，目标对象通过场景标签和空间位置消歧。

### 1:40–2:05 验证缓存

说：

> 重复上次描述。

如果场景上下文相同，页面显示“验证缓存命中”；强调缓存内容是已经经过 Pydantic 和 Guardrails 的 DrawingPlan，而不是未校验的模型原文。

### 2:05–2:30 中断与回滚

发起复杂任务，在逐步绘制时说：

> 停止。

说明系统使任务 token 失效并恢复任务前快照，避免半成品。

### 2:30–2:50 恢复能力

说：

> 保存作品。

再说明页面刷新后也可说“恢复现场”，恢复本机自动保存点。

### 2:50–3:00 总结

“项目覆盖纯语音闭环、三秒静默、多候选容错、三路执行、复杂拆解、安全 DSL、事务回滚、作品持久化和赛前自检，并提供完整设计文档与自动化证据。”
