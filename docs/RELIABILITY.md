# 比赛现场稳定性设计

## 1. 语音输入稳定性

- SpeechRecognition 最多读取 5 个候选；
- 候选按浏览器置信度、绘图领域关键词、文本长度和噪声词惩罚综合排序；
- “嗯、啊、哦、那个、然后”等单独出现时不会进入绘图描述；
- 短文本只有在浏览器明确给出极低置信度时才被拒绝，兼容不提供置信度的浏览器；
- 相同 final 片段在短时间重复返回时自动去重；
- 语音服务网络错误采用指数退避重连，避免持续高频重启。

## 2. 三秒静默稳定性

- interim/final 文本和 Web Audio VAD 都会刷新最后语音活动时间；
- 每次活动递增版本号，旧定时器不能提交新一轮文本；
- 未记录到有效内容时不会启动自动提交；
- “立即绘图”可以人工跳过等待，“继续描述”可以显式延续输入。

## 3. AI 请求稳定性

- 全局复用 `httpx.AsyncClient`，减少重复 TCP/TLS 建连；
- 连接超时与总超时分离；
- 429 和 5xx 按 `Retry-After` 或指数退避进行有限重试；
- JSON 校验失败时只进行一次结构修复；
- 成功方案进入有界 TTL/LRU 缓存；
- 缓存只保存已校验、已 Guardrails 清洗的 DrawingPlan，并在读取时深拷贝。

## 4. 画布事务安全

- 每个任务开始前保存快照；
- 停止、网络错误、请求超时、客户端 DSL 白名单失败或执行异常都会回滚；
- 前端只允许 clear/background/create/delete/recolor/transform；
- create 只允许九种白名单图元；
- 每次成功操作后写入按用户隔离的本机恢复点。

## 5. 赛前检查清单

1. 使用 `scripts/start_competition_kiosk.bat` 启动；
2. 允许浏览器麦克风权限；
3. 说“系统自检”；
4. 确认页面显示“全部就绪”；
5. 先演练复杂创作，再演练本地编辑和停止回滚；
6. 准备备用网络，且提前让常用演示指令生成一次以建立验证缓存；
7. 公开仓库和展示电脑中不得存在真实 API Key 的 Git 记录或截图。
