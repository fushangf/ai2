# 评分点与实现证据映射

| 题目评分点 | 实现 | 现场演示证据 |
|---|---|---|
| 纯语音控制 | 开始、停止、撤销、重做、保存、恢复、帮助、自检均为语音命令 | 首次授权后全程不碰鼠标键盘 |
| 结束判定 | Web Audio VAD + 识别结果 + 版本锁，静默 3 秒自动提交 | 页面音量条与倒计时实时变化 |
| 指令准确性 | 多候选打分、置信度展示、近音归一化、噪声过滤、重复片段去重 | 故意加入“嗯、那个”或说“开使” |
| 对象消歧 | id/group_id/label/tags/bbox 联合匹配，支持最左、最右、第二个、最大、所有 | 对三个同类云朵指定“第二个”或“最右边” |
| 容错性 | JSON 自动修复、Guardrails、网络重试、超时回滚、崩溃恢复 | 说“停止”或断网后展示画面未被破坏 |
| 响应延迟 | 本地快速编辑、验证缓存、连接池、系统命令本地化 | 重复复杂指令显示“缓存命中”；编辑显示“本地低延迟” |
| 复杂拆解 | execution_steps 与多操作 DSL | 一句话执行 transform/recolor/delete |
| 可解释性 | 页面展示执行路径、拆解步骤、DSL、延迟和识别置信度 | 打开 DSL 面板并讲解 |
| 安全性 | 不执行模型代码、颜色/坐标/ID限制、客户端二次白名单、账号限流 | 展示 Pydantic、Guardrails 与客户端校验 |
| 工程完整性 | 登录、数据库、管理员、作品、自动恢复、Docker、测试 | 管理后台与 21 项测试报告 |
| 设计文档 | 计划、完成、未完成原因和稳定性策略齐全 | `docs/DESIGN.md`、`docs/RELIABILITY.md` |
