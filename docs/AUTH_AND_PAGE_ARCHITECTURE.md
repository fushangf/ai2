# 登录与页面架构说明（4.3）

## 页面路由

| 路由 | 页面 | 说明 |
|---|---|---|
| `/` | 产品落地页 | 展示产品能力、技术架构和使用流程 |
| `/login` | 用户登录 | 默认打开用户登录选项卡 |
| `/register` | 用户注册 | 完全独立的新注册页面 |
| `/workspace` | 用户工作台 | 纯语音绘图与语音交流模型 V2 |
| `/admin/login` | 管理员登录 | 复用登录门户并默认打开管理员选项卡 |
| `/admin/dashboard` | 管理后台 | 总览、用户管理、使用统计和系统状态 |

`/app` 和 `/admin` 仍保留兼容跳转，分别跳转到 `/workspace` 和 `/admin/dashboard`。

## 角色隔离

- 普通用户登录：`POST /api/login/user`
- 管理员登录：`POST /api/login/admin`
- 兼容旧客户端：`POST /api/login`

角色专用接口会验证账号角色。普通用户不能通过管理员入口登录，管理员账号也不会进入普通用户工作台。

## 注册流程

注册页面只创建普通用户。注册成功后自动签发 Token，并跳转到 `/workspace`。管理员账号只能通过初始化配置创建，避免公开注册管理员。

## 安全设计

- 密码使用 PBKDF2 哈希存储；
- JWT Token 包含用户 ID、用户名和角色；
- 受保护接口再次从数据库读取用户状态；
- 被封禁账号无法重新登录或继续调用受保护接口；
- 管理后台接口要求管理员角色；
- 公共代码仓库不包含真实 API Key、数据库密码或管理员密码。
